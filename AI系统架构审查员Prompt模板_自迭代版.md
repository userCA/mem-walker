## 使用说明

### 自迭代原理

本模板内置自省机制，每次评审执行后会自动：
1. **收集执行数据**（耗时、发现数、评分）
2. **评估有效性**（与历史基线对比）
3. **触发迭代条件检测**
4. **自动修改模板**（如满足条件）

### 执行流程

```
第1步: 读取【内置状态区】获取当前版本
第2步: 执行评审，填充各维度评分
第3步: 填写【执行记录区】
第4步: 【内省分析器】评估是否触发迭代
第5步: 如触发迭代 → 自动修改模板 → 版本号+1
第6步: 更新【内置状态区】
```

## 模板自迭代使用指南

### 快速开始

1. **首次使用**：直接执行第一部分评审，填写所有维度
2. **执行完成**：填写第二部分汇总
3. **执行记录**：填写第三部分执行记录
4. **内省分析**：执行第四部分自检
5. **迭代决策**：
   - 如需迭代 → 填写第五部分 → 更新第六部分状态 → 版本号+1
   - 如无需迭代 → 仅更新第六部分状态

### 迭代生效机制

```
修改第五部分内容后，模板文件本身即被更新。
下次执行时，新版本模板自动生效。
无需额外操作。
```

### 有效性保障

| 指标 | 目标 | 监控方式 |
|------|------|---------|
| 迭代采纳率 | >70% | 第六部分历史记录 |
| 评审完整性 | >90% | 第四部分自检 |
| 版本可控性 | 版本号连续 | 状态区记录 |

### 注意事项

- **保持JSON格式**：状态区必须保持有效JSON格式
- **版本号语义**：主版本变更需人工确认
- **反馈积累**：避免重复反馈淹没真正问题
- **定期复盘**：每10次执行做一次深度复盘
