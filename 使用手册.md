# Mnemosyne 记忆系统使用手册

本文档基于 `service/examples/verify_memory_system.py` 的验证流程，详细说明当前高性能记忆系统的实例化、配置与调用方法。

## 1. 系统架构与组件
Mnemosyne 记忆系统采用模块化设计，核心组件包括：
- **Embedding**: 负责文本向量化 (FastEmbed)。
- **Vector Store**: 负责向量存储与检索 (Milvus)。
- **LLM**: 负责实体提取、事实推理与冲突检测 (SimpleLocalLLM/Qwen)。
- **Graph Store**: 负责知识图谱存储 (Graph/Neo4j)。
- **Reranker**: 负责混合检索结果的重排序 (BM25)。

## 2. 快速开始

### 2.1 环境依赖
确保以下服务已启动：
1.  **Docker Milvus**: 向量数据库 (默认端口 19530)。
2.  **Local LLM Service**: 兼容 OpenAI 接口的本地模型服务 (如 vLLM/Ollama 运行 Qwen，默认端口 8000)。

### 2.2 核心组件实例化

```python
import os
from mnemosyne.embeddings.fastembed import FastEmbedEmbedding
from mnemosyne.embeddings.configs import FastEmbedConfig
from mnemosyne.vector_stores.milvus import MilvusVectorStore
from mnemosyne.vector_stores.configs import MilvusConfig
from mnemosyne.llms.simple_local_llm import SimpleLocalLLM
from mnemosyne.reranker.bm25 import BM25Reranker
from mnemosyne.reranker.configs import RerankerConfig
# from mnemosyne.graphs.neo4j import Neo4jGraphStore # 生产环境使用

# 1. Embedding (本地快速向量化)
embed_config = FastEmbedConfig(
    model="BAAI/bge-small-en-v1.5",
    dimension=384
)
embedding = FastEmbedEmbedding(embed_config)

# 2. Vector Store (Milvus)
vector_store_config = MilvusConfig(
    collection_name="mnemosyne_memory",
    vector_size=384,
    host=os.getenv("MILVUS_HOST", "localhost"),
    port=int(os.getenv("MILVUS_PORT", 19530)),
    index_type="FLAT"  # 测试用 FLAT，生产建议 HNSW
)
vector_store = MilvusVectorStore(vector_store_config)

# 3. LLM (本地低延迟模型)
# 自动读取环境变量 LOCAL_LLM_BASE_URL 和 LOCAL_LLM_MODEL
llm = SimpleLocalLLM()

# 4. Reranker (BM25)
reranker = BM25Reranker(RerankerConfig())

# 5. Graph Store (图存储)
# 这里使用 DummyGraphStore 作为示例，实际请配置 Neo4j
from mnemosyne.graphs.base import GraphStoreBase
class DummyGraphStore(GraphStoreBase):
    # ... 实现接口 ...
    pass
graph_store = DummyGraphStore()
```

### 2.3 记忆系统初始化

将所有组件注入 `Memory` 主类：

```python
from mnemosyne.memory.main import Memory

memory = Memory(
    embedding=embedding,
    vector_store=vector_store,
    graph_store=graph_store,
    llm=llm,
    reranker=reranker
)
```

## 3. 核心功能调用

### 3.1 添加记忆 (Add)

保存用户的对话或输入内容。

```python
user_id = "test_user"
content = "Mnemosyne 是一个高效的记忆系统。"

# 方式一：完整模式 (推荐)
# 包含：Embedding + 向量存储 + LLM 事实提取 + 图谱更新
# 耗时：约 1.2s (Local LLM)
memory.add(content, user_id=user_id, infer=True)

# 方式二：快速模式
# 包含：Embedding + 向量存储
# 耗时：< 0.05s
memory.add(content, user_id=user_id, infer=False)
```

### 3.2 搜索记忆 (Search)

根据查询语句检索相关记忆。系统内部会自动并行执行向量检索和图谱实体提取，以实现最低延迟。

```python
query = "什么是 Mnemosyne?"

# 执行混合搜索
# 流程：
# 1. 并行执行: [Query Embedding -> Vector Search] & [LLM Entity Extraction -> Graph Expansion]
# 2. 结果合并与重排序 (Rerank)
results = memory.search(query, user_id=user_id)

# 耗时：约 0.27s (Local LLM + 并行优化)
for res in results:
    print(res)
```

## 4. 性能指标 (参考)

基于 `verify_memory_system.py` 的实测数据 (Mac M1/M2 + Local Docker Milvus + Local Qwen):

| 操作阶段 | 耗时 (约) | 说明 |
| :--- | :--- | :--- |
| **Add (Full)** | 1.16s | 包含 LLM 推理与存储 |
| **Add (Fast)** | 0.03s | 纯向量写入 |
| **Search** | 0.27s | 端到端查询耗时 |
| *Vector Search* | 0.02s | 向量检索部分 |
| *Entity Extract*| 0.31s | 实体提取 (并行执行，不阻塞主流程) |

```env
LOCAL_LLM_MODEL=qwen2.5-7b-instruct
LOCAL_LLM_BASE_URL=http://localhost:8000/v1
```
